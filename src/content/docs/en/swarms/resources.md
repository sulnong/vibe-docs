## Use resources by question type

Official docs are best for supported behavior. Package metadata is best for requirements. Source code is best for exact implementation. Issues are useful for symptoms and edge cases.

| Need | Start with |
| --- | --- |
| Install or version requirement | [PyPI package](https://pypi.org/project/swarms/) |
| Supported usage | [Official docs](https://docs.swarms.world/) |
| Implementation detail | Repository source. |
| Recent rough edges | Repository issues and changelog. |

## Keep local notes short

Project notes should record decisions: version pin, provider, enabled workflows, tool contracts, output schema, deployment path, and rollback. Do not mirror an entire official docs site.
