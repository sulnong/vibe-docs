# AI, ML, and Model Platforms Skills

Skills for model APIs, ML platforms, training workflows, and AI product integration.

Use this category when you know the broad job but still need to decide which skill is worth invoking. Start from the task, then check the source, dependency, first run, and exit conditions.

## How to choose

- Build against fast-moving model APIs with provider-specific constraints.
- Run ML platform workflows without losing track of credentials and artifacts.
- Compare model or platform choices through concrete tasks.

## Avoid this category when

- The task needs current model pricing or policy and the source has not been checked.
- Sensitive datasets would be uploaded without review.
- The expected result is scientific validation rather than workflow guidance.

## Decision table

| Check | Guidance |
| --- | --- |
| Main dependency | Model provider account, API key, dataset, or SDK context |
| Best first use | Use a small example request, dataset, or model operation before production workflows. |
| Smallest safe scope | Start with a low-risk example, preview environment, small file, or read-only target so the result can be reviewed. |
| When to switch | If the task falls outside this category's real boundary, go back to the [Awesome Skills overview](/en/awesome-skills/) and choose a closer category. |
