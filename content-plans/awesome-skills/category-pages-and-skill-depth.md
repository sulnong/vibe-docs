# Awesome Skills 分类页与 Skill 页面加厚记录

## 本轮目标

- 为第一轮 50 个 skills 增加 8 个分类页，形成「topic 首页 -> 分类页 -> skill 页」的三层结构。
- 让侧边栏按分类展示，每个分类下先放「分类概览」，再放该分类的具体 skill。
- 在非样页 skill 页面里补齐读者决策信息，而不是只给来源和安装入口。

## 分类页结构

每个分类页回答四类问题：

- 这个分类适合什么类型的任务。
- 该分类下有哪些 skills，各自最适合处理什么。
- 使用前要满足哪些依赖和边界条件。
- 如果当前分类不适合，应该回到哪里选择相邻分类。

分类页主要承担 SEO 聚合页和读者导航页的作用。它们可以承接更宽泛的搜索词，例如 browser automation skills、AI model platform skills、agent security review skills，也能把权重内链到具体 skill 页面。

## Skill 页面加厚方向

每个非样页 skill 页面新增以下公开读者可读内容：

- 这页帮你判断什么。
- 什么时候使用、什么时候不要使用。
- 决策清单，包括依赖、最小试运行、验收方式和停止条件。
- 与同分类相邻 skill 的对比。
- 第一次工作流、使用边界、相似 skill 和参考资料。

这些内容避免使用「搜索意图」「差异化」「页面目标」等内部策划词，公开页面只保留读者可直接使用的判断和说明。

## SEO 意义

- 首页覆盖 Awesome Skills 主题词和总目录需求。
- 8 个分类页覆盖中等宽度的任务型关键词。
- 50 个 skill 页覆盖长尾关键词、具体工具名、发布方名和安装/使用问题。
- 分类页与 skill 页互相内链，能减少孤立页面，并帮助搜索引擎理解 topic 结构。

后续如果继续扩到 100 个 skills，优先补每个分类页的真实案例、失败边界、竞品对比和常见错误信息。
