# Hermes topic 改写记录

## 背景

Hermes 公开页面已经有基础覆盖，但多数页面像从大纲直接生成：段落短、重复多、判断少，读者打开后很难形成“我下一步该怎么做”的路径感。当前公开路由使用 `/hermes-agent/`，对品牌词与短查询不够友好；站点首页也没有指向 Hermes，搜索引擎和普通读者都不容易发现。

## 本次调整

- 公开路由从 `/en/hermes-agent/`、`/zh/hermes-agent/` 改为 `/en/hermes/`、`/zh/hermes/`。
- 保留旧路由 301 到新路由，避免已有链接直接断掉。
- 将 Hermes topic 重写成读者可用的技术文档：先解释使用场景、安装路径、第一次运行、模型配置、工具边界，再进入 memory、skills、gateway、cron、delegation、MCP、API、架构和排错。
- 站内链接统一改为 `/hermes/` 前缀。
- 站点首页加入 Hermes 入口，提升内部链接可发现性。

## 写作原则

- 不把内部选题词、搜索意图、差异化、人工审核等字段写入公开页面。
- 每页回答一个真实读者问题，避免堆同一套小标题。
- 命令和配置给上下文，不孤立列清单。
- 涉及远程入口、自动化、工具执行时，把安全边界和回滚路径写清楚。
- 英文和中文保持同一信息结构，但不逐字硬翻。

## 后续建议

- 发布后在 sitemap 和 Search Console 中提交 `/en/hermes/`、`/zh/hermes/`。
- 观察是否出现 `hermes agent install`、`pip install hermes-agent`、`Hermes Agent tutorial`、`Hermes Agent vs Claude Code` 等查询；若有展现但点击低，再按具体查询改标题和描述。
- 如果旧 `/hermes-agent/` 已被索引，保留 301 至少数月。
