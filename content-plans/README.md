# 选题大纲报告目录

`content-plans/` 只存 agent 为某个选题生成的研究与大纲报告。它不是人工维护的队列，也不是需要用户手动跑的工作流后台。

正常流程：

1. 用户在 agent 中输入一个 topic。
2. agent 尝试 clone 官方仓库；如果耗时太久就放弃，改用 GitHub API、README、raw 文件和官方文档。
3. agent 查找更多与选题相关的官方文档、release notes、examples、issues、FAQ、迁移指南、源码示例、社区问题、竞品页面和 SERP 结果。
4. agent 抽取事实，标注来源，整理例子、坑点、对比、排错路径和搜索需求。
5. agent 生成 `content-plans/<topic-slug>/outline.md`，供人工审核。
6. 用户确认大纲报告后，agent 再生成正式中英文文档。

大纲报告必须包含：

- 研究流程与资料来源。
- URL 路径设计，包括英文和中文入口。
- 页面组织方式，包括哪些页面、页面顺序和页面之间的关系。
- 每个页面要回答的问题、搜索意图、内容结构、来源需求和差异化内容。
- 人工审核问题，帮助决定是否进入文档生成阶段。

这个目录里的报告可以更新，但不要把它变成选题状态机、GSC 指标仓库或人工命令手册。
