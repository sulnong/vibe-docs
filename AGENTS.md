# Project Instructions

- We are in a fast iteration phase. Do not run tests, builds, or verification commands unless the user explicitly asks for them.
- When making changes, prefer small, reviewable edits that keep the local dev server easy to inspect.
- Public pages should stay topic-first. Do not expose internal workflow, queue, factory, or style-preview pages to readers.
- 所有文档都用中文。
- 讨论、产出、plan、设计都要形成精简文档，作为需求对齐的重要参考。
- 不特意强调时，不用建新分支，默认在 `main` 分支上操作。
- 不要生成 `.superpowers` 目录；如果出现，应删除。
